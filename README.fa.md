# داشبورد سئو — ردیاب رتبه کلمات کلیدی

یک داشبورد خودمیزبان برای رهگیری رتبه کلمات کلیدی در چند سایت مختلف،
با استفاده از **API رایگان Google Search Console**. یک ادمین، چند پروژه، تم تاریک/روشن، رابط کاربری انیمیشن‌دار.

> **English version:** [README.md](README.md)

---

## این پروژه چیست و چه فایده‌ای دارد؟

### سئو چیست؟

وقتی کسی توی گوگل سرچ می‌کند مثلاً «بلیط هواپیما کیش»، گوگل لیستی از سایت‌ها نشان می‌دهد.
**سئو (SEO)** کارهایی است که انجام می‌دهید تا سایتتان در این لیست بالاتر بیاید.
هرچه بالاتر باشید، آدم‌های بیشتری روی سایتتان کلیک می‌کنند، بازدید بیشتر، مشتری بیشتر.
جایگاه سایت در نتایج گوگل را **رتبه (Position/Rank)** می‌گویند — رتبه ۱ یعنی اول لیست.
**عدد کمتر بهتر است.**

### Google Search Console چیست؟

ابزاری **رایگان از خود گوگل** که می‌گوید سایتتان در گوگل چه وضعیتی دارد:

- مردم با **چه کلماتی** سرچ کردند و سایتتان را دیدند
- سایتتان برای هر کلمه **چه رتبه‌ای** دارد
- چند نفر سایتتان را **دیدند** (Impression) و چند نفر **کلیک** کردند (Click)

مشکل Search Console: رابط کاربری خامی دارد — تاریخچه روند نشان نمی‌دهد، وقتی رتبه افت می‌کند هشدار نمی‌دهد، و امکان مقایسه چند سایت همزمان ندارد.

### این داشبورد دقیقاً چه می‌کند؟

هر روز ساعت ۴ صبح به‌صورت خودکار به Search Console وصل می‌شود،
دیتای ۳۰ روز اخیر سایت‌هایتان را می‌کشد، در دیتابیس محلی ذخیره می‌کند و اینها را نشان می‌دهد:

| بخش | چه نشان می‌دهد | فایده |
|---|---|---|
| **Overview** | میانگین رتبه، کلیک، نمایش، نمودار روند | یک نگاه کلی: سایتم دارد بهتر می‌شود یا بدتر؟ |
| **Keywords** | همه کلماتی که باهاشان در گوگل دیده می‌شوید + رتبه هر کدام | می‌فهمید برای چه کلماتی رتبه دارید و کدام‌ها مهم‌اند |
| **Movers & Drops** | کلماتی که رتبه‌شان زیاد بالا یا پایین رفته | سریع می‌فهمید کجا پیشرفت کردید، کجا افت کردید |
| **Mobile** | مقایسه رتبه موبایل با دسکتاپ | گوگل به موبایل اهمیت بیشتری می‌دهد — اگه موبایل ضعیف است می‌فهمید |
| **Alerts** | هشدار خودکار وقتی کلمه‌ای افت شدید می‌کند | لازم نیست هر روز چک کنید — خودش خبرتان می‌کند |

## امکانات

- **Overview** — تعداد کلمات Top 10 / Top 20 / Top 50، میانگین رتبه، نمودار توزیع
- **مقایسه بازه زمانی** — این هفته در برابر هفته قبل / این ماه در برابر ماه قبل (رتبه، Top 3/10، کلیک، نمایش، CTR)
- **امتیاز Visibility** — یک عدد کلی سلامت سئو (سهم دیده‌شدن) با نمودار روند ۳۰/۹۰ روزه
- **یادداشت‌ها (Annotations)** — تغییرات سایت را با تاریخ ثبت کنید و اثرشان را به‌صورت خط‌نشان روی همه نمودارها ببینید
- **All Keywords** — جدول کامل با مرتب‌سازی، فیلتر، رتبه دسکتاپ/موبایل، تغییر ۷ روزه، نمودار کوچک
- **نمودار روند هر کلمه** — روی هر کلمه کلیک کنید تا نمودار ۳۰/۹۰ روزه دسکتاپ + موبایل باز شود
- **نمای صفحات (Pages)** — عملکرد هر URL: چند کلمه می‌آورد، بهترین/میانگین رتبه، کلیک، روند
- **Insights** — نمودار کلیک/نمایش، فرصت‌های CTR (نمایش بالا + کلیک کم = تایتل/توضیحات را بهینه کن) و تشخیص هم‌نوع‌خواری کلمات (Cannibalization)
- **Movers & Drops** — کلماتی که هفته‌به‌هفته صعود یا نزول کرده‌اند
- **Mobile** — مقایسه کامل رتبه موبایل و دسکتاپ با ستون GAP
- **سلامت سایت (Site Health)** — وضعیت ایندکس هر صفحه (URL Inspection API) + امتیاز PageSpeed و Core Web Vitals
- **داشبورد عمومی مشتری** — لینک گزارش فقط‌خواندنی بدون لاگین، با قابلیت ابطال فوری
- **Alerts** — هشدار خودکار برای هر جابجایی بیش از ۵ رتبه
- **Live Sheet** — صدور یک‌کلیکه به Google Sheets (کلمات، موورز، تاریخچه کامل)
- **صدور CSV**
- **سینک خودکار روزانه** ساعت ۴ صبح
- تم تاریک/روشن (با سیستم هماهنگ)، انیمیشن Framer Motion
- چندپروژه — هر تعداد سایت که بخواهید

## راه‌اندازی قدم‌به‌قدم (از صفر تا اولین سینک)

### قدم ۱ — سایتتان را به Search Console اضافه کنید

قبل از سینک، سایت باید با همان اکانت گوگلی که به داشبورد وصل می‌کنید **تأیید شده** باشد.

**گزینه الف — Domain Property (توصیه‌شده)**

همه ساب‌دامین‌ها و هر دو http/https را پوشش می‌دهد.
رشته GSC Property در داشبورد: `sc-domain:yourdomain.com`

1. به [search.google.com/search-console](https://search.google.com/search-console) بروید
2. دراپ‌داون بالا-چپ → **Add property**
3. **Domain** را انتخاب کنید، `yourdomain.com` بنویسید → **Continue**
4. گوگل یک رکورد TXT مثل `google-site-verification=XXXXX` می‌دهد
5. وارد پنل DNS دامنه‌تان شوید (Cloudflare، Namecheap، …) و یک رکورد **TXT** جدید با آن مقدار اضافه کنید
6. در Search Console **Verify** بزنید (روی Cloudflare معمولاً آنی است، روی سایرین ممکن است چند ساعت طول بکشد)

**گزینه ب — URL-prefix Property**

فقط همان URL دقیقی که ثبت می‌کنید را پوشش می‌دهد. تأیید راحت‌تر (فایل HTML یا meta tag).
رشته GSC Property: `https://yourdomain.com/` (اسلش آخر اجباری است)

### قدم ۲ — سطح دسترسی را چک کنید

API Search Analytics نیاز به دسترسی **Owner** یا **Full User** دارد.
دسترسی **Restricted User** در رابط وب GSC کار می‌کند اما API را مسدود می‌کند — خطای زیر را می‌گیرید:

```
User does not have sufficient permission for site '...'
```

برای بررسی: Search Console → سایت را انتخاب کنید → **Settings** → **Users and permissions**

| سطح دسترسی | دسترسی به API | چطور ایجاد می‌شود |
|---|---|---|
| **Owner** | بله | خودتان سایت را تأیید کرده‌اید |
| **Full User** | بله | مالک سایت شما را با این سطح اضافه کرده |
| **Restricted User** | **خیر** | وقتی کسی شما را اضافه می‌کند بدون مشخص کردن سطح |

برای ارتقا: روی منوی سه‌نقطه کنار ایمیلتان کلیک کنید → **Change permission** → **Full** را انتخاب کنید.

### افزودن سایت مشتری یا شخص دیگر

برای ردیابی سایت مشتری **نیازی به پسورد گوگل او ندارید** و **لازم نیست اکانت گوگل
جداگانه‌ای بسازید**. داشبورد فقط با **یک** اکانت گوگل (مال خودتان) وصل می‌شود و هر
پراپرتی‌ای که آن اکانت به آن دسترسی داشته باشد به‌صورت خودکار نمایش داده می‌شود —
حتی اگر سایت متعلق به شخص دیگری باشد.

پس برای ردیابی سایت مشتری، کافی است مشتری به اکانت گوگل شما دسترسی پراپرتی
Search Console خودش را بدهد:

۱. مشتری وارد [Search Console](https://search.google.com/search-console) می‌شود و
   پراپرتی **خودش** را انتخاب می‌کند.
۲. مسیر **Settings** → **Users and permissions** → **Add user**.
۳. **همان ایمیل گوگلی که شما به این داشبورد وصل کرده‌اید** را وارد می‌کند.
۴. سطح دسترسی را روی **Full** بگذارد (توصیه‌شده) تا API کار کند.
   حالت *Restricted* فقط اجازه دیدن داده در رابط وب را می‌دهد و API را مسدود می‌کند.
۵. روی **Add** کلیک می‌کند.

همین! هیچ تغییری در کد لازم نیست و هیچ لاگین اضافه‌ای نمی‌خواهد. در داشبورد، داخل
پنجره Add Project دکمه **Refresh** را بزنید تا سایت مشتری در دراپ‌داون پراپرتی‌ها
ظاهر شود و آماده افزوده‌شدن به‌عنوان پروژه جدید باشد.

> **چرا یک اکانت برای چند سایت کافی است؟** دسترسی‌های Search Console بر اساس
> پراپرتی تعریف می‌شوند، نه بر اساس اکانت. یک اکانت گوگل می‌تواند هم‌زمان به ده‌ها
> سایت مختلف از افراد مختلف دسترسی Owner/Full داشته باشد. داشبورد در حال حاضر یک
> اتصال گوگل واحد را ذخیره می‌کند، پس همه پروژه‌ها از طریق همان یک اکانت مجاز
> خوانده می‌شوند.

### قدم ۳ — OAuth گوگل را تنظیم کنید

[docs/GOOGLE_SETUP.md](docs/GOOGLE_SETUP.md) را دنبال کنید تا اعتبارنامه‌های OAuth را بسازید و متغیرهای `GOOGLE_*` را در `.env` پر کنید.

### قدم ۴ — گوگل را در داشبورد وصل کنید

1. وارد داشبورد شوید
2. **Connect Google** را در هدر کلیک کنید
3. با همان اکانت گوگلی که به Search Console دسترسی دارد وارد شوید
4. مجوزها را تأیید کنید (خواندن داده Search Console + ساختن/ویرایش Sheets)

### قدم ۵ — پروژه اضافه کنید

1. روی **Add Project** در صفحه اصلی کلیک کنید
2. نام و دامنه را بنویسید
3. در قسمت **GSC Property** دراپ‌داونی می‌بینید که از اکانت متصل‌تان پر شده — فقط سایت‌هایی که Owner یا Full User هستید نشان داده می‌شوند
4. پراپرتی را انتخاب کنید و **Create Project** بزنید

### قدم ۶ — سینک کنید

**Sync Now** را روی کارت پروژه بزنید. اولین سینک ۳۰ روز داده می‌کشد (~۳۰ درخواست API، یک تا دو دقیقه ممکن است طول بکشد). بعد از این، cron ساعت ۴ صبح همه چیز را خودکار به‌روز نگه می‌دارد.

## راهنمای خواندن داده‌ها

### عدد کمتر = رتبه بهتر

رتبه ۱ یعنی اول لیست. GSC میانگین می‌دهد پس اعشار مثل ۳.۴ طبیعی است.

| رتبه | رنگ |
|---|---|
| ۱–۳ | سبز |
| ۴–۱۰ | آبی |
| ۱۱–۲۰ | نارنجی |
| ۲۱–۵۰ | خاکستری |
| ۵۱+ | قرمز |

### تأخیر در داده‌های GSC

داده‌های Search Console **۲–۳ روز تأخیر** دارند. سینک امروز داده‌ها را تا تقریباً پریروز می‌کشد.
این رفتار طبیعی گوگل است — همه ابزارهای ردیابی رتبه که بر GSC بنا شده‌اند همین تأخیر را دارند.

### کدام کلمات بیشترین فرصت را دارند؟

کلماتی با رتبه **۵ تا ۲۰** بهترین فرصت هستند: یک بهبود کوچک در محتوا می‌تواند آن‌ها را به صفحه اول بیاورد و کلیک‌هایتان را چند برابر کند.

## شروع سریع (توسعه محلی)

```bash
cp .env.example .env          # مقادیر را پر کنید
make install                  # npm install + prisma generate
npx prisma migrate dev        # ساخت دیتابیس SQLite
make hash-password PASSWORD=YourPassword123   # خروجی را در .env کپی کنید
make dev                      # http://localhost:3000
```

داده نمونه بدون اتصال به گوگل: `npx prisma db seed`

برای استقرار روی سرور: [docs/INSTALLATION.md](docs/INSTALLATION.md)
